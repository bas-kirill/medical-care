## Сервиса генерации QR кодов

Сервис генерирует и хранит QR-код и основную информацию о нем, которую вычитывает из сообщений Kafka. Вместо реального QR кода генерируем хэш по алгоритму MD5 от полученных из Kafka данных. Далее под QR кодом подразумеваем именно этот хэш. Сервис выставляет REST API для получения данных о QR коде пользователя и его валидации.

Требования к API:

* GET ```/qr/{passport}```: выдает последний самый свежий код пользователю для последующего предъявления по его паспорту;
* GET ```/qr/check?сode={code}```: принимает QR-код, удостоверяет его подлинность и актуальность (тем самым подтверждает полномочия владельца);

Дополнительные требования:

* На каждое уникальное событие вакцинации должен быть создан ровно один QR код.
* Сервис обеспечивает персистентное хранение QR кодов в БД.
* Сервис зарегистрирован в service registry

Сборка JAR: ```mvn clean compile assembly:single```

Технический стек: Spring Boot, Spring MVC, Spring Kafka
