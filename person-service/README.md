## Person Service.

Сервис хранит личную информацию о гражданах: ФИО, паспортные данные, другие документы, телефоны, адреса и прочее.

Информация о каждом человеке вносится через сотрудников поддержки.

У одного гражданина могут быть:
* Несколько документов (паспорт, загранпаспорт, СНИЛС, страховой полис)
* Несколько адресов проживания и один адрес регистрации
* Несколько контактов (телефон, email)

Сущности:
* ```Person``` - представление гражданина
* ```IdentityDocument``` - [OneToMany] разные документы, удостоверяющие личность, работа с enum
* ```Address``` - [ManyToMany]
* ```Contact``` - контакты [OneToMany]


Что надо сделать:

* Реализовать доменную модель. Создать таблицы, сущности и закрепить связи между ними.
* Создать слой репозиториев для основных запросов к этим сущностям
* Создать сервисные методы, вызывающие репозитории
* REST API:
  * Создание пользователя
  * Получение пользователя по ID
  * Проверить пользователя по паспорту и полному имени
  * Обновить сведения о пользователе
  * Удалить пользователя
* Предусмотреть обработку исключений
* Настроить логирование запросов/ответов

Опционально:
* Добавить возможность применять миграции при изменении схемы данных с помощью liquibase / flyway / pgmigrate.

Сборка JAR: make build-jar<br>
Запуск сервиса: make run

Технический стек: Spring Boot, Spring Data, Spring MVC, Spring Cloud, Testcontainers, JUnit
